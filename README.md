- Unity URP 로 제작된 개인 프로젝트입니다.
- 모바일과 PC 빌드이며 직장인들의 점심 메뉴를 3D 캐릭터들의 달리기 경주로 결정하는 게임입니다.
- 참가선수들의 이름을 결정하고 경기에 참여시킬 수 있으며, 각각 속도나 컨디션에 관련된 파라미터가 유기적으로 작동하도록 세팅하였습니다.
- 캐릭터들은 웨이포인트 기반 이동경로를 따라 이동하며 중간경로에 도착하면 다음 경로를 탐색하는 방식으로 트랙이 구성되어 있습니다.
- 이름 타입으로 지정된 메뉴들이 얼마나 우승했는지 로컬로 저장합니다.

## 2. 개발환경
- UNITY 2020.3.18f1 / URP / C#
- Visual Studio 2019
- Windows 10
- 2025.01 ~ 2026.04 (16개월)
- 개발 1명 (1인개발)

## 3. 사용기술
- 디자인 패턴 : Single Tone 패턴으로 매니저 관리
- Postprocessing : 퀄리티업을 위해 다양한 후처리 효과가 적용되었습니다.
- Textmeshpro : 고품질 UI 텍스트 및 말풍선에 적용되었습니다.
- Timeline : 시네마틱 애니메이션 연출에 사용되었습니다.
- MashBaker : 드로우콜 최적화를 위해 3D 오브젝트의 매쉬를 통합하였습니다.
- ToonRender : 3D 오브젝트에 툰 셰이더를 적용하였습니다.

## 4. 기능구현
- 캐릭터 AI 이동 시스템  
    웨이포인트 네비게이션: 체크포인트 배열을 순서대로 추적하며 트랙을 따라 이동합니다.  
    캐릭터의 능력치 및 실시간으로 변동되는 컨디션, 외부적인 요인에 따라 속도가 가중됩니다.  
    모두 같은 경로를 따라 이동하면 어색하기에 3개 이상의 레인을 배치하였으며 무작위로 변경됩니다.  
- 캐릭터 스탯 시스템  
    Condition / Power / Endirance 능력치가 특정 상황에 따라 속도를 계산하는 방식으로 구현하였습니다.  
    캐릭터 능력치는 init에서 일정 범위내로 무작위로 결정됩니다.  
- 상태 머신 (러닝 루틴)  
    스프린트, 조깅, 걷기, 비틀거림 상태가 레이스 중 부여되며 각 특징에 맞게 애니메이션이 동기화 되어있습니다.  
- 실시간 순위  
    버블 정렬 알고리즘으로 0.25초 인터벌 마다 순위가 재 계산됩니다.  
    완주 시 결승점을 통과한 순서대로 최종 순위가 매겨지며 화면에 표시됩니다.  
- 카메라 시스템  
    레이스중인 임의 대상을 Vector3.Lerp + Quaternion.Lerp로 부드럽게 따라가며 8초 간격으로 대상을 전환합니다.  
    체크포인트 잔여 3개 시점인 피니시 시퀀스에선 카메라 구도가 극적으로 변환되며 슬로우모션이 발동됩니다.  
- 데이터  
    PlayerPrefs 기반 로컬 저장으로 우승자 히스토리 배열을 저장합니다.
